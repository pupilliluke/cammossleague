# Cloud Deployment Guide for Cam Moss League

## Option 1: Railway (Recommended - Easiest)
**Best for: Full-stack deployment with PostgreSQL**

### Setup:
1. Go to https://railway.app/
2. Connect your GitHub repo
3. Railway auto-detects Spring Boot + PostgreSQL
4. Get $5/month free credit

### Commands:
```bash
# Install Railway CLI
npm install -g @railway/cli

# Login and deploy
railway login
railway link
railway up
```

### Environment Variables in Railway:
- `DATABASE_URL` - Auto-generated by Railway PostgreSQL
- `DATABASE_USERNAME` - Auto-generated  
- `DATABASE_PASSWORD` - Auto-generated
- `CORS_ALLOWED_ORIGINS` - Your frontend URL

---

## Option 2: Render + Neon PostgreSQL
**Best for: Free hosting with external database**

### Render (Backend):
1. Go to https://render.com/
2. Connect GitHub → Deploy Spring Boot
3. Free tier available

### Neon (Database): 
1. Go to https://neon.tech/
2. Create free PostgreSQL database (512MB)
3. Get connection string

---

## Option 3: Vercel Frontend + Railway Backend
**Best for: Keeping frontend on Vercel**

### Setup:
1. **Frontend**: Deploy React app to Vercel
2. **Backend**: Deploy Spring Boot to Railway
3. **Database**: Railway PostgreSQL or Neon

### Vercel Configuration (Frontend Only):
```json
{
  "builds": [
    {
      "src": "client/package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "dist"
      }
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/client/$1"
    }
  ],
  "env": {
    "VITE_API_URL": "https://your-backend.railway.app/api"
  }
}
```

## Option 4: Heroku (Classic)
**Best for: Traditional deployment**

### Setup:
```bash
# Install Heroku CLI
# Create app
heroku create cam-moss-league
heroku addons:create heroku-postgresql:mini

```

## Current Vercel.json Issues:

Your current `vercel.json` tries to deploy both frontend and backend to Vercel, which has limitations:

```json
{
  "builds": [
    {
      "src": "pom.xml",
      "use": "@vercel/java"  // ❌ Limited Spring Boot support
    }
  ]
}
```

## Recommended: Split Architecture

### Frontend (Vercel):
- React app
- Fast CDN
- Perfect for static sites

### Backend (Railway/Render):
- Spring Boot API  
- PostgreSQL database
- Better for Java apps

This gives you the best of both platforms!
